# Dark Forge Autonomous Loop

This loop adapts the strongest part of the `3650` process:

- trust the rendered site more than the code
- critique failures before praising strengths
- use screenshot evidence to drive each revision

Working sequence:

1. Review current-site screenshots.
2. Write a blunt critique with concrete failures.
3. Define the intended visual system and page structure.
4. Build or revise the prototype.
5. Re-capture desktop and mobile screenshots.
6. Repeat until the design feels sharper, stranger, and more deliberate than the live site.

Minimum critique format per iteration:

- `What fails`
- `What is acceptable but weak`
- `What is genuinely working`
- `What to change next`
- `Confidence check`
